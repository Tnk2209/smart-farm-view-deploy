ระบบพยากรณ์และเตือนภัยของกรมการข้าว
๔.๒ พัฒนาระบบในรูปแบบ Web Application สามารถใช้งานได้ในรูปแบบ Web Responsive
รองรับการใช้ผ่าน Web Browser Chrome,Microsoft Edge และ Firefox ได้เป็นอย่างน้อย
๔.๓ พัฒนากระบวนการเตือนภัย ด้วยการใช้ข้อมูลจากสถานีตรวจวัดอากาศอัตโนมัติ ข้อมูลจากการ
สำรวจแปลง ข้อมูลจากดาวเทียม ข้อมูลองค์ความรู้ด้านการผลิตข้าว เป็นอย่างน้อย
๔.๔ ระบบสามารถแจ้งเตือนความเสี่ยงการเกิดภัยพิบัติ การระบาดของโรคและแมลงศัตรูข้าว
ตามที่กรมการข้าวกำหนด ในแบบอัตโนมัติและแบบแมนนวลในรูปแบบ Push Notification
๔.๕ ระบบต้องสามารถนำเสนอข้อมูลในรูปแบบ Visualization และ Dashboard เพื่อนำข้อมูล
มาแสดงผลได้ในรูปแบบต่างๆได้ เช่น แผนภูมิ, รูปภาพ และตาราง เป็นอย่างน้อย
๔.๖ ระบบต้องสามารถแสดงผลข้อมูล (Dashboard) จำนวนของเครื่องมือวัดและ อุปกรณ์รวบรวมข้อมูล
ส่งสัญญาณทางไกล ทั้งหมดในโครงการได้
๔.๗ ระบบต้องสามารถแสดงผลข้อมูล (Dashboard) โดยต้องสามารถเลือกช่วงเวลาของข้อมูลที่นำมา
แสดงผลได้ตามความต้องการ
๔.๔ ระบบต้องสามารถจัดทำระบบรายงานโดยสามารถ Export ออกมาได้ในรูปแบบของ Excel
และสามารถระบุช่วงเวลาที่ต้องการข้อมูลได้
๔.๙ ระบบต้องมีเครื่องมือแสดงข้อมูลของเครื่องมือวัดและอุปกรณ์รวบรวมข้อมูลส่งสัญญาณทางไกลได้
เพื่อแสดงผลข้อมูลจากสถานนี้ตรวจอากาศรวมถึงสามารถดูข้อมูลย้อนหลังของอุปกรณ์ได้
๔.๑๐ ระบบต้องสามารถแจ้งเตือนสถานการณ์ภัยพิบัติด้านข้าวโดยมีรายละเอียดอย่างน้อยดังนี้
a) ความเสี่ยงภัยแล้ง
b) ความเสี่ยงน้ำท่วม
C) ความเสี่ยงพายุ
d) ความเสี่ยงการระบาดของโรคและแมลง
๔.๑๑ สามารถกำหนดระบบงานหรือเป้าหมายในการแจ้งเตือนในแต่ละครั้งได้ ผ่านหน้า Web Ul
โดยกำหนดเป้าหมายการแจ้งเตือนได้ในระดับจังหวัด ตำบล อำเภอ หรือการแจ้งเตือนรายเกษตรกร/
กลุ่มเกษตรกร ได้เป็นอย่างน้อย และสามารถกำหนดเงื่อนไขการแจ้งเตือนอื่น ๆ ได้ตามที่กรมการข้าวกำหนด
๔.๑๒ ระบบต้องสามารถวิเคราะห์และรายงานสถานการณ์ความเสี่ยงภัยพิบัติด้านข้าว พร้อมการแจ้งเตือน
ความเสี่ยงการเกิดภัยพิบัติ การระบาดของโรคและแมลงศัตรูข้าวแบบอัตโนมัติและแบบแมนนวล
ในรูปแบบ Push Notification
๔.๑๓ ระบบต้องรองรับการใช้ข้อมูลจากสถานีตรวจวัดอากาศ ข้อมูลจากการสำรวจแปลง ข้อมูลจาก
ดาวเทียม ข้อมูลองค์ความรู้ด้านการผลิตข้าว และข้อมูลสถานการณ์ภัยพิบัติจากการเชื่อมโยงกับหน่วยงานอื่น
ในการวิเคราะห์สถานการณ์
๔.๑๔ ระบบต้องสามารถแสดงผลสถานการณ์ความเสี่ยงการเกิดภัยพิบัติ โรคและการระบาดของแมลงศัตรูข้าว
บน Application ALLRice
๔.๑๕ มีการจัดเก็บ Log การแจ้งเตือนทุกครั้งในรูปแบบ Time Series Database พร้อมจัดทำหน้าเมนู
สำหรับ แสดงผลและสืบค้น ข้อมูล Logs โดยจะต้องมีการจัดเก็บข้อมูลอย่างน้อยดังนี้
วันเวลาที่ส่งแจ้งเตือน
หัวข้อที่ทำการแจ้งเตือน
เป้าหมายที่ทำการแจ้งเตือน
ระบบ หรือ ผู้ทำการแจ้งเตือน
๔.๑๖ ระบบสามารถออกรายงานแสดงรายละเอียดการแจ้งเตือน จำนวนครั้งที่มีการแจ้งเตือน
ระบบที่ได้รับการแจ้งเตือน ได้เป็นอย่างน้อย โดยสามารถ Export รายงานได้ในรูปแบบ Excel หรือ PDF ได้
๔.๑๗ พัฒนาส่วนการแสดงผลสถานการณ์ความเสี่ยงการเกิดภัยพิบัติ โรคและการระบาดของแมลงศัตรูข้าว
สำหรับแสดงผลบน Application ALLRice
๔.๑๘ ระบบต้องรองรับการกำหนดกลุ่มเป้าหมายในการแจ้งเตือน โดยสามารถระบุกลุ่มเป้าหมาย
ของการแจ้งเตือนได้ ทั้งระดับจังหวัด ตำบล อำเภอ หรือการแจ้งเตือนรายเกษตรกร/กลุ่มเกษตรกร
